## STEP 1｜建立 Google Sheet（訂單記錄試算表）

### 1-1 新增試算表
1. 前往 https://sheets.google.com
2. 點擊「空白」→ 新增試算表
3. 命名為：**美和科大中餐團購訂單**

### 1-2 工作表設定
第一個工作表（Sheet1）重命名為「**訂單記錄**」

> GAS 部署後會自動建立標題列，**不需要手動輸入欄位名稱**

### 欄位說明（系統自動建立）

| 欄 | 欄位名稱 | 說明 |
|----|---------|------|
| A | 時間戳記 | 訂單送出的完整日期時間 |
| B | 日期 | yyyy-MM-dd 格式 |
| C | 訂購人姓名 | 填寫的姓名 |
| D | 部門/班級 | 填寫的部門或班級 |
| E | 聯絡電話 | 選填 |
| F | 店家名稱 | 訂餐的餐廳 |
| G | 餐點名稱 | 品項名稱 |
| H | 規格/選項 | 大/小、湯/乾、麵條種類等 |
| I | 數量 | 訂購數量 |
| J | 單價 | 每份單價 |
| K | 小計 | 數量 × 單價 |
| L | 訂單總額 | 同一訂單第一筆填入總金額 |
| M | 備註 | 訂購人的備註 |

## STEP 2｜建立 Google Apps Script

### 2-1 從試算表開啟 GAS
1. 在 Google Sheet 中，點選上方選單：**擴充功能 → Apps Script**
2. 進入 Google Apps Script 編輯器

### 2-2 貼上程式碼
1. 刪除編輯器中的預設程式碼（`function myFunction() {}`）
2. 將 `美和科大午餐系統.gs` 的全部內容**貼上**
3. 點擊上方磁碟圖示或按 **Ctrl+S** 儲存
4. 專案名稱命名為：**美和科大午餐系統**

### 2-3 測試功能
1. 在 GAS 編輯器中，從下拉選單選擇函數 **`testSystem`**
2. 點擊 ▶ 執行
3. 第一次執行會要求授權：
   - 點擊「審查權限」
   - 選擇你的 Google 帳戶
   - 點擊「進階」→「前往 美和科大午餐系統（不安全）」
   - 點擊「允許」
4. 確認：
   - ✅ Google Sheet 出現測試訂單資料
   - ✅ 兩個 Email 信箱收到測試通知信

### 2-4 設定每日彙整觸發器
1. 選擇函數 **`setupTrigger`**
2. 點擊 ▶ 執行
3. 系統自動設定「每天下午 14:00 發送每日彙整 Email」
4. 可在 GAS 左側 **時鐘圖示（觸發條件）** 確認設定

## STEP 3｜部署為 Web API

### 3-1 部署步驟
1. 點擊右上角 **「部署」→「新增部署作業」**
2. 點擊「類型」旁的齒輪 ⚙️ → 選擇 **Web 應用程式**
3. 填寫設定：

| 設定項目 | 值 |
|---------|-----|
| 說明 | 美和科大午餐系統 v1.0 |
| 執行身份 | **我（你的帳號）** |
| 誰可以存取 | **所有人** |

4. 點擊「**部署**」
5. **複製 Web 應用程式網址（URL）**
   格式如：`https://script.google.com/macros/s/AKfyc.../exec`

### 3-2 驗證 API
在瀏覽器中直接貼上 URL，應顯示：
```json
{
  "status": "ok",
  "system": "美和科技大學中餐團購系統",
  "message": "美和科大中餐團購系統運作正常 ✅"
}
```

## STEP 4｜更新前端 HTML

### 4-1 替換 GAS URL
打開 `meiho_lunch.html`，找到第幾行：

```javascript
const GAS_URL = 'https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec';
```

將 `YOUR_SCRIPT_ID` 替換為真實的部署 URL：

```javascript
const GAS_URL = 'https://script.google.com/macros/s/AKfycXXXXXXXXXX/exec';
```

## STEP 5｜部署至 GitHub Pages

### 5-1 建立 GitHub 帳號與 Repository
1. 前往 https://github.com 並登入（或註冊帳號）
2. 點擊右上角 **「+」→「New repository」**
3. 填寫設定：

| 設定 | 值 |
|------|-----|
| Repository name | `meiho-lunch` |
| 說明 | 美和科技大學中餐團購系統 |
| Public/Private | **Public**（Pages 免費版需 Public） |
| Initialize | ✅ Add a README file |

4. 點擊「**Create repository**」

### 5-2 上傳 HTML 檔案

**方法A：網頁直接上傳（最簡單）**
1. 在 Repository 頁面，點擊「**Add file → Upload files**」
2. 將修改好的 `meiho_lunch.html` 拖曳上傳
3. 重要：**將檔名改為 `index.html`** 才能作為首頁
4. 填寫 Commit message：`Add lunch ordering system`
5. 點擊「**Commit changes**」

**方法B：Git 指令（進階）**
```bash
git clone https://github.com/你的帳號/meiho-lunch.git
cd meiho-lunch
cp /path/to/meiho_lunch.html index.html
git add index.html
git commit -m "Add lunch ordering system"
git push origin main
```

### 5-3 啟用 GitHub Pages
1. 在 Repository 頁面，點擊「**Settings**」
2. 左側選單找到「**Pages**」
3. Source 選擇：
   - Branch: **main**
   - Folder: **/ (root)**
4. 點擊「**Save**」
5. 等待約 1-2 分鐘，重新整理頁面
6. 頁面頂部會出現：

```
Your site is live at https://你的帳號.github.io/meiho-lunch/
```

### 5-4 測試部署
打開上述網址，確認：
- ✅ 系統正常顯示
- ✅ 五間店家分頁可切換
- ✅ 可加入餐點至購物車
- ✅ 送出訂單後 Email 收到通知

## STEP 6｜更新版本（日後維護）

每次修改 HTML 後，GAS 也需要重新部署：

### 更新 HTML（GitHub）
```bash
git add index.html
git commit -m "Update menu - 2024/XX/XX"
git push origin main
```
GitHub Pages 約 1 分鐘後自動更新。

### 更新 GAS（後端）
1. 修改程式碼後，點「**部署 → 管理部署作業**」
2. 點擊鉛筆圖示 ✏️ 編輯
3. 版本選擇「**建立新版本**」
4. 點擊「**部署**」（URL 不變）

## 🔧 常見問題排解

### Q1：送出訂單後沒有收到 Email
**解決方法：**
1. 確認 GAS 部署時「執行身份」設為「我」
2. 確認「誰可以存取」設為「所有人」
3. 在 GAS 執行 `testSystem()` 確認授權正常

### Q2：試算表沒有資料寫入
**解決方法：**
1. 確認 HTML 中的 `GAS_URL` 已更新為正確 URL
2. 在瀏覽器直接訪問 GAS URL，確認顯示 `{"status":"ok",...}`
3. 檢查 GAS 的「執行項目」log 是否有錯誤訊息

### Q3：CORS 錯誤
系統使用隱藏 form + iframe 方式提交，可避免 CORS 問題。
確認 HTML 的送出方式使用 `form.submit()` 而非 `fetch()`。

### Q4：GitHub Pages 沒有更新
- GitHub Pages 有快取，請等待 2-5 分鐘
- 強制重整：按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）
